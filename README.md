# Mentor-Matching: алгоритм рекомендации в сервисе поиска и подбора менторов

## Описание проекта

Цель проекта — разработать и протестировать алгоритм рекомендации для сопоставления менторов и менти в условиях отсутствия реальных пользовательских данных. Поскольку сервис еще не запущен, в качестве основы используются синтетические профили, сгенерированные по заранее заданной структуре.

В проекте решаются следующие задачи:
1. генерация профилей менторов и менти
2. реализация алгоритма рекомендации
3. анализ причин, по которым рекомендации выдаются
4. тестирование алгоритма при разной степени заполненности профиля

## Идея подхода

В качестве базового решения используется content-based подход с rule-based scoring. Это означает, что рекомендация строится не на истории взаимодействий пользователей, а на сопоставлении характеристик профиля ментора и менти.

Алгоритм учитывает:
- совпадение профессиональной области
- пересечение навыков
- соответствие целей менти экспертизе ментора
- совместимость по уровню
- совпадение предпочитаемого формата взаимодействия
- совпадение языка общения
- совпадение индустриального интереса.

Такой подход выбран по следующим причинам:
- отсутствие исторических данных
- необходимость объяснимых рекомендаций
- возможность протестировать решение на синтетических профилях
- удобство анализа устойчивости алгоритма при неполных данных

## Структура проекта

- `src/profile_schema.py` — словари допустимых значений признаков
- `src/data_generation.py` — генерация синтетических профилей
- `src/matching.py` — логика расчета recommendation score
- `src/explainability.py` — объяснение выданных рекомендаций
- `src/evaluation.py` — эксперименты и метрики
- `main.py` — запуск генерации, матчинга и экспериментов
- `report/draft_report.md` — текст отчета

## Логика recommendation score

Итоговый score для пары mentor–mentee рассчитывается как взвешенная сумма частных совпадений:

Score = w1 * domain_match + w2 * skills_overlap + w3 * goal_match + w4 * seniority_fit + w5 * format_fit + w6 * language_fit + w7 * industry_fit

Где:
- `domain_match` — совпадение профессиональной области
- `skills_overlap` — степень пересечения навыков
- `goal_match` — соответствие целей менти темам и экспертизе ментора
- `seniority_fit` — подходит ли ментор по уровню
- `format_fit` — совпадение по формату взаимодействия
- `language_fit` — совпадение языка
- `industry_fit` — совпадение интереса к индустрии

## Эксперименты

В рамках проекта предусмотрены следующие эксперименты:
1. проверка базового качества рекомендаций на полных профилях
2. анализ объяснимости рекомендаций
3. тестирование при разной заполненности профил
4. анализ чувствительности результатов к весам признаков.

## Практическая ценность

Разработанный алгоритм может использоваться как baseline-решение для ранней версии сервиса mentor-matching до появления реальных пользовательских данных и фидбэка.
